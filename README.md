## 1) 为什么“只换IP不够”？（3分钟理解）
中文：平台判断关联不只看 IP，还会看浏览器指纹、存储、字体、时区、分辨率、插件等。稳定的方案是：**独立浏览器环境 + 稳定代理 + 规范流程**。
English: Platforms don’t rely on IP alone. They also look at fingerprint, storage, fonts, timezone, resolution, plugins, etc. A stable setup is: **isolated profiles + consistent proxy + clean workflow**.

---

## 2) 新手最稳配置步骤（Beginner Setup Steps）
**Step 1**：为每个账号创建独立浏览器 Profile（不要共用）
**Step 2**：每个 Profile 绑定一个“稳定代理”（不要频繁切换）
**Step 3**：时区/语言/分辨率与代理地区保持一致
**Step 4**：账号资料与登录行为固定（不要今天A环境明天B环境）
**Step 5**：需要协作时，用“共享环境/权限”而不是共享账号密码
**Step 6**：记录环境命名规则（平台-用途-地区-日期）

(EN)
**Step 1**: Create one isolated profile per account (never reuse)
**Step 2**: Bind one stable proxy per profile (avoid frequent switching)
**Step 3**: Keep timezone/language/resolution consistent with proxy region
**Step 4**: Keep login behavior consistent (same account, same profile)
**Step 5**: For teams, share profiles/permissions instead of passwords
**Step 6**: Use a clear naming convention (platform-use-region-date)

---

## 3) 最常见的“关联点”清单（Common Linking Points）
- 一个浏览器登录多个账号 / multiple accounts in one profile
- 频繁换 IP/地区 / frequent IP or region switching
- 本地缓存/存储混用 / shared local storage/cookies
- 时区语言与IP地区不一致 / mismatch between timezone/language and IP region
- 团队多人共用账号密码 / sharing passwords across teammates

## 4) 选择建议（按场景选）

English: If you need profile management + team collaboration + isolation, choose tools with strong profile grouping, permissions, and import/export.
